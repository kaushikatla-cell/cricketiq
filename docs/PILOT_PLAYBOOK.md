# 4-week pilot playbook (CricketIQ)

Use this lightweight playbook to run a credible pilot with your school or club group and capture **measurable** outcomes for essays, interviews, and sponsor conversations.

## Week 0 — Setup (30 minutes)

- [ ] Create a season in **Seasons** (name it after your term or tournament).
- [ ] Add your roster in **Players**.
- [ ] Optionally load sample data once to learn the UI, then reset when you are ready for real entries.

## Week 1 — Baseline

- [ ] Log every match played this week (minimum viable: runs, balls, wickets, catches).
- [ ] Record **attendance** (how many unique players showed up) in a notebook or spreadsheet.
- [ ] Ask every player one question: “What would make practices feel fairer?” (write down themes)

## Week 2 — Rhythm

- [ ] Keep match logging consistent (same person or rotating “stat keeper”).
- [ ] Review **Rankings** and **Best XI** in a 10-minute group huddle: focus on roles, not “who is bad.”
- [ ] Capture one quote from a teammate about what surprised them in the data.

## Week 3 — Strategy

- [ ] Use **Predictor** before a friendly internal scrimmage; compare prediction vs outcome.
- [ ] Export CSV backups from the dashboard (players + matches).
- [ ] Run **Impact Report** and screenshot or print to PDF for your portfolio.

## Week 4 — Reflection

- [ ] Re-measure attendance and compare to Week 1 (honest numbers beat a perfect story).
- [ ] Run a 3-question pulse survey (Google Form is fine):
  1. Do you feel selections are clearer than four weeks ago? (1–5)
  2. Did CricketIQ change how you think about improvement? (short text)
  3. What should we build next? (short text)

## Metrics cheat sheet

| Metric | Where to get it |
|--------|------------------|
| Players tracked | Impact Report / dashboard |
| Matches recorded | Impact Report / dashboard |
| Runs / wickets analyzed | Impact Report |
| Season standings | Seasons page |

## Ethics reminder

Analytics should **support** players, not shame them. Use CricketIQ to celebrate improvement, clarify roles, and make selection criteria transparent.
